# 段永平投资分析 Skill

## 描述
基于段永平（大道）在网易博客和雪球上的公开问答录原文，运用他的投资理念和商业逻辑对目标股票进行**层层递进式综合分析**。

**v2.0 核心升级**：新增双路径决策模型（生意优先型/创始人优先型）、创始人深潜独立维度、迭代认知循环。
**v3.0 核心升级**：新增环境变化检测层——在引用任何段永平过去判断之前，先检查条件是否已变；结构性颠覆时走"框架外思考"路径。

## 核心框架

本 skill 使用 `dy/knowledge/` 下的知识架构，核心是**双路径决策模型**：

### 第一步：环境变化检测（Step 0）

在分析任何公司之前，先扫描颠覆性变化——这是整个框架的"地基"。

```
□ 技术颠覆：过去3-5年这个行业被AI/新技术改写了吗？
□ 消费者行为变化：用户和10年前还一样吗？
□ 竞争格局变化：有新进入者带着不同模式打进来吗？
□ 段永平历史判断的前提条件检查：
   "段永平说这个公司不错的时候，环境是什么样的？现在变了吗？"

如果检测到"结构性颠覆"→ 不硬套框架
  走"框架外思考"：承认不确定性，用小仓位观察，等待认知升级
```

### 第二步：判断公司类型（Step 0.5）

在开始分析之前，先判断这家公司属于哪种类型：

```
Path A — 生意优先型（茅台、苹果）：
  生意模式自明，护城河结构性强，换CEO不影响
  → 从商业模式入手分析

Path B — 创始人优先型（泡泡玛特、早期苹果）：
  模式还在演化，创始人是理解生意的钥匙
  → 从创始人深潜入手分析
```

### 条件逻辑链

**第零关：环境变化检测（新增——所有分析的前提）**
```
颠覆性信号扫描完成？ → ☑ 无颠覆 ☑ 改良性变化 ☑ 结构性颠覆
如果结构性颠覆 → 不硬套框架，走"框架外思考"路径
```

**Path A（生意优先型）：**
```
第零关通过 → 第一关：商业模式（40%，一票否决权）
    ↓ 通过
第二关：企业文化（35%，决定持续性）
    ↓ 通过  
第三关：价格/估值（25%，入场时机）
    ↓
实地验证 → 综合判断 → 10年视角 → 迭代认知循环
```

**Path B（创始人优先型）：**
```
第零关通过 → 创始人深潜（前置判断）
    ↓ 通过
第一关：商业模式（35%，通过创始人视角重新审视）
    ↓ 通过
第二关：企业文化（35%，验证文化是否制度化）
    ↓ 通过
第三关：价格/估值（30%，B型确定性低，需更好价格补偿）
    ↓
实地验证（重点验证创始人叙事和现实的匹配度）
    → 综合判断 → 10年视角 → 迭代认知循环
```

## 分析流程（九步法）

### Step 0：环境变化检测（新增）
扫描颠覆性信号——技术变革、消费者行为变化、竞争格局变化。
引用段永平任何过去判断前，先检查前提条件是否还在。

### Step 0.5：公司类型判断
判断公司是生意优先型（A）还是创始人优先型（B）。

### Step 1：排除测试
回答四个问题，任何一项不通过则直接跳过：
1. 这个生意我真正懂吗？
2. 生意模式一眼就能看出好吗？
3. 管理层有不本分迹象吗？
4. 有致命Stop Doing违规吗？

### Step 2：生意叙事
写出100-300字的"生意逻辑描述"——写不出来就是没懂。

### Step 3：深度分析（根据类型选路径）
**Path A：** 商业模式 → 企业文化 → 价格
**Path B：** 创始人深潜 → 商业模式（重审）→ 企业文化 → 价格

### Step 4：实地验证
亲自验证关键假设——用产品、看门店、聊用户、观竞品。

### Step 5：综合判断
三个维度的方向是否一致？条件逻辑链是否全部通过？

### Step 6：10年视角验证
- 10年后护城河更宽还是更窄？
- 如果股价跌一半，想买还是想跑？
- "能睡好觉吗？"

### Step 7：迭代认知循环
设置下次检视信号，准备修正认知。

## 引用原则

- 所有引用的段永平观点必须出自 `dy/knowledge/` 目录下的原文
- 权重体系标注为"基于段永平原话的归纳"，不是段永平本人的精确分类
- 分析报告中的个人推论需与段永平原话明确区分
- 引用案例时优先使用 `06-案例思维拆解.md` 中的思维过程拆解

## 特殊分析模式

### 快速筛选模式
用户输入"快速筛选 [公司名]"，执行 Step 0 + Step 1，判断是否值得进一步分析。

### 反事实分析模式
用户输入"如果[公司名]股价跌了一半/[某个条件变化]，段永平会怎么做"，运用段永平的决策逻辑进行推演。

### 迭代跟踪模式
用户输入"帮我更新对[公司名]的判断"，用新数据验证原有分析，执行迭代认知循环。

## 报告输出格式

**所有分析报告必须使用 HTML 格式输出**，不得使用 Markdown 格式。

### 使用方式

1. 从 `dy/reports/template.html` 读取 HTML 模板
2. 将 `REPORT_TITLE` 替换为报告标题（格式如：`哔哩哔哩（BILI / 9626.HK）段永平方法论综合分析报告`）
3. 将 `ANALYSIS_DATE` 替换为当前分析日期
4. 将 `CURRENT_PRICE` 替换为当前股价/市值信息（如无可省略该 meta-item 整行）
5. 将 `REPORT_BODY` 替换为完整的报告正文 HTML 内容
6. 保存到 `dy/reports/` 目录下，文件名：`{公司名}分析报告.html`

### HTML 样式指南

报告正文中使用以下 CSS 类来保持样式一致：

| 用途 | 推荐样式 |
|------|---------|
| **章节标题** | `<h2 class="section-title">` + `<span class="step-badge">Step N</span>` |
| **子标题** | `<h3 class="section-subtitle">` |
| **表格** | 标准 `<table>` 含 `<thead>` 和 `<tbody>`，包裹在 `<div class="table-wrapper">` 中 |
| **判断结论** | 使用 `<span class="tag tag-green/red/yellow">` |
| **通过/不通过** | `<span class="verdict-pass">✅ 通过</span>` / `<span class="verdict-fail">❌ 不通过</span>` |
| **结果框** | `<div class="result-box pass/fail/warn/info">` |
| **引用段永平原话** | `<blockquote>` |
| **代码块/逻辑链** | `<div class="code-block">` — 支持 `.pass` `.fail` `.warn` `.dim` `.highlight` 子样式 |
| **检查清单** | `<ul class="checklist">` + `<li><span class="icon pass/fail/warn">✅/❌/⚠️</span>` |
| **卡片信息** | `<div class="card">` + `<h4>` 标题 |
| **维度对比** | `<div class="dim-row">` + `<div class="dim-card">` |
| **分隔线** | `<hr>` |

### 内容转换规则

| Markdown 原始写法 | HTML 对应写法 |
|-----------------|--------------|
| `## 标题` | `<h2 class="section-title">` |
| `### 子标题` | `<h3 class="section-subtitle">` |
| `| 表格 \|` | `<div class="table-wrapper"><table>` |
| `> 引用` | `<blockquote>` |
| `` ``` `` 代码块 | `<div class="code-block">` |
| `- [ ] / - [x]` 清单 | `<ul class="checklist">` |
| `---` | `<hr>` |
| `**粗体**` | `<strong>` |
| `✅ / ❌ / ⚠️` 等图标 | 保持原样或用对应语义类 |

### 示例

请参考 `dy/reports/哔哩哔哩分析报告.md` 的内容结构，将其转换为 HTML 格式输出。保持内容结构不变（Step 0→Step 7 的九步法流程），仅将格式从 Markdown 替换为 HTML。
