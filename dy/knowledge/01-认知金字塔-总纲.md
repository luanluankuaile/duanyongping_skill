> ⚠️ **本文件是整个知识库的"操作系统"。** 分析任何公司之前先读这篇，理解完整决策框架，再用后面的维度文件做深度分析。

> 段永平说"未来现金流折现只是一种思维方式，千万不要去套公式"——同样，**框架是思维辅助，不是机械指令**。

## 一、核心升级：双路径决策模型

本次重构最重要的发现：段永平的分析不是一条路径走到底，而是**根据公司类型选择不同的起点**。

### 公司类型的两种画像

| 特征 | **A型：生意优先型** | **B型：创始人优先型** |
|------|-----------------|-------------------|
| 生意模式 | 自明性好，一眼能看出来好不好 | 模式还在演化中，需要深入理解才能判断 |
| 护城河来源 | 品牌/专利/网络效应/规模等结构性壁垒 | 创始人的视野和执行是核心变量 |
| 创始人的重要性 | 重要但可替代（造钟人已就位） | 至关重要（目前还是报时人，或在转变为造钟人途中） |
| 典型代表 | 茅台、苹果（库克时代）、可口可乐 | 泡泡玛特、早期苹果（乔布斯时代）、网易（段永平投时） |
| 错误分析方法 | 不适用 | **用标准清单打分 → 会完全错过本质** |

## 二、公司类型判断（Step 0）——选路径

在开始任何分析之前，先判断这家公司是A型还是B型。

### 诊断问题

```
□ 这个生意的"好"是显而易见的，还是需要深入理解创始人才能看到？
   → 显而易见 → 偏向A型（生意优先）
   → 需要理解创始人才看到 → 偏向B型（创始人优先）

□ 如果创始人明天离职，公司3年后会怎样？
   → 基本不变 → A型（造钟已完成）
   → 大幅下滑 → B型（创始人仍是核心变量）

□ 公司的护城河是结构性的（品牌/专利/网络效应），
   还是依赖于创始人的视野和判断？
   → 结构性 → A型
   → 依赖创始人 → B型

□ 公司的生意模式在成熟行业中还是新兴/正在演化的行业中？
   → 成熟 → A型
   → 新兴/演化中 → B型
```

### 判断示例

| 公司 | 类型 | 判断依据 |
|------|------|---------|
| 茅台 | 生意优先型 | 品牌护城河自明，换CEO不影响 |
| 苹果（库克时代） | 生意优先型 | 造钟已完成，制度驱动 |
| 苹果（乔布斯时代） | 创始人优先型 | 当时乔布斯是核心变量 |
| **泡泡玛特** | **创始人优先型** | **模式还在演化，王宁是理解生意的钥匙** |
| 网易（段永平投时） | 创始人优先型 | 段永平的判断基于自己对行业的理解，而非公共信息 |

> **如果判断错了会怎样？**
> - 把B型当A型分析 → 会错过本质（如用标准清单分析泡泡玛特，只会看到"IP集中度风险"而看不到王宁的价值）
> - 把A型当B型分析 → 浪费大量时间在创始人研究上，错过了生意本身的确定性

## 三、创始人深潜（新增独立维度）

### 3.1 为什么需要独立？

原来的框架把创始人评估放在"企业文化(35%)"下的一个子项。**对于B型公司，这远远不够。**

段永平的投资泡泡玛特的决策过程揭示了：**他是先看懂了王宁，才重新理解了泡泡玛特的生意。**

> **"我理解王宁不是因为我的投资，而是因为我曾经是个企业家，我能看懂他有多厉害。"**

> **"王宁对商业的理解好像比Jobs还要强一点点。"**

这不是"企业文化评分"能捕捉的判断。

### 3.2 创始人深潜方法论

对于B型公司，在进入商业模式分析之前，先做创始人深潜：

#### 第一步：传记研究

```
找到创始人最详实的外部记录：
  □ 公开传记/访谈录（如《因为独特》——王宁）
  □ 深度媒体专访（如李翔对王宁的长篇对话）
  □ 创始人在公开场合的演讲、写作
  □ 创始人在公司发展关键节点上的决策记录

目标不是搜集信息，而是理解：
  • 创始人是如何思考问题的？
  • 他在面临艰难选择时如何做决策？
  • 他对"好产品"和"好生意"的定义是什么？
  • 他的认知在过去几年有进化吗？
```

#### 第二步：企业家共情（段永平最独特的能力）

段永平说"我理解王宁是因为我曾经是个企业家"。这不是一般人能复制的，但可以接近：

```
  你不需要做过企业家才能评估创始人，但你需要问自己：
  
  □ 如果让我来做这个生意，我能不能做出这个创始人做出的决定？
  □ 他在高速增长面前选择降速——这是克制还是缺乏信心？
  □ 他在行业第一时自评组织管理70分——这是坦诚还是作秀？
  □ 他的产品理念是我相信的，还是只是听起来好？

  核心：把自己代入创始人的位置，看自己能不能做得更好。
  如果你发现"换了我绝对做不出他这样的决定"→ 这是好信号。
```

#### 第三步：创始人分类

段永平评估创始人时，实际上有几种原型：

| 类型 | 特征 | 段永平评价 | 案例 |
|------|------|-----------|------|
| **造钟人** | 建制度、建文化、培养接班人 | "库克是乔布斯最伟大的发现" | 库克 |
| **产品天才** | 对产品有极致追求和理解 | "对产品的理解与Jobs一个级别" | 王宁、乔布斯 |
| **本分人** | 坚守原则、不做不对的事 | "马化腾人不错"、"江南春比较本分" | 马化腾、黄峥 |
| **年轻有为** | 还有很长的任期，"复利是吓人的" | "他还能至少好好干25年以上" | 王宁、扎克伯格 |

**注意：这些类型不互斥，一个创始人可以同时具备多个。**

#### 第四步："企业家复利"概念（新增）

> **"他还能至少好好干25年以上。这个复利是吓人的。"**

段永平引入了"复利"思维来看待创始人：
- 一个好创始人每多干一年，能力、认知、行业资源都在增长
- 如果他还年轻，这种复利效应能持续20-30年
- 这意味着：**创始人的年龄是投资决策中的一个重要变量**

在分析中加入：**创始人的"剩余任期" × 每年的认知增长 = 企业家复利**

### 3.3 创始人深潜的输出

完成深潜后，应该能回答：

```
□ 我对这个创始人的理解度：高 / 中 / 低
□ 如果我是他，我做不做得出今天泡泡玛特这些决策？→ 能/不能
□ 创始人的认知在过去2-3年有进化吗？→ 有/没有
□ 他在"Stop Doing"上的表现：有原则/没表现/有问题
□ 他的"企业家复利"：年轻+强=高复利 / 年长+强=中等 / 不确定
□ 总的来说：他值得我信任吗？
```

## 四、更新后的条件逻辑链

### Path A：生意优先型（标准路径）

```
Step 1: 排除测试（通过？→ 继续）
    ↓
Step 2: 生意叙事（初步理解）
    ↓
Step 3a: 商业模式深度分析（40%权重）
    ↓ 通过
Step 3b: 企业文化诊断（35%权重）
    ↓ 通过  
Step 3c: 价格/估值（25%权重）
    ↓
Step 4: 实地验证（关键判断验证）
    ↓
Step 5: 综合判断 + 10年视角
    ↓
Step 6: 进入迭代认知循环（监测异常信号）
```

### Path B：创始人优先型（新增路径）

```
Step 1: 排除测试（通过？→ 继续）
    ↓
Step 2: 生意叙事（初步理解——此时可能还不深）
    ↓
Step 3a: 创始人深潜（新增独立维度）
    ↓ 通过（理解了创始人）
Step 3b: 通过创始人视角重新审视商业模式
    ↓ 现在能看懂了
Step 3c: 企业文化诊断（验证创始人的文化是否制度化）
    ↓
Step 3d: 价格/估值
    ↓
Step 4: 实地验证（重点验证"创始人叙事"是否和"现实"匹配）
    ↓
Step 5: 综合判断 + 10年视角
    ↓
Step 6: 进入迭代认知循环
```

### 核心差异对比

| 环节 | A型：生意优先 | B型：创始人优先 |
|------|-------------|---------------|
| 深度分析的起点 | 商业模式 | 创始人 |
| 商业模式评估方式 | 直接用框架评估 | 通过创始人视角理解后再评估 |
| 创始人在评估中的角色 | 文化维度的一部分 | **独立的第一关** |
| 价格容忍度 | 确定性来自生意本身 | 确定性来自对创始人的信任 |
| 主要风险 | 护城河变窄、竞争加剧 | 创始人出问题、接班人缺失 |

## 五、权重体系更新：双模式权重

### Path A：生意优先型（权重不变）

```
商业模式（40%） → 一票否决权
企业文化（35%） → 持续性保证
价    格（25%） → 入场时机

说明：同原框架（已在上面部分详细推导过，此处不重复）
```

### Path B：创始人优先型（权重调整）

```
创始人（独立评估，不设百分比，但决定其他维度的诠释方式）
     ↓
商业模式（35%）→ 在理解了创始人之后重新评估
企业文化（35%）→ 验证创始人的文化是否制度化
价    格（30%）→ B型公司确定性较低，需更好的价格补偿

说明：
  • "创始人深潜"不设具体百分比，因为它的作用不是"打分"
    而是决定了你如何理解其他维度
  • 价格权重从25%升到30%，因为B型公司的确定性通常低于A型
  • 商业模式从40%降到35%，不是因为生意不重要，而是因为
    需要通过创始人视角才能准确评估——所以创始人是前置判断
```

### 权重不是公式（再次强调）

> **权重是思维辅助，不是数学公式。** 段永平自己说没人用公式做投资决策。

权重帮你回答的问题：
- 当两个维度冲突出以哪个为准？
- 当需要压缩分析时间时，优先保哪个？
- 投资泡泡玛特时，为什么段永平先研究王宁再看生意？

## 六、实地验证（新增步骤）

段永平投资泡泡玛特的关键行动之一：

> **"不大的门店生意极好，进店顾客绝大多数是本地成年人，华人占比仅一成左右。"**

### 为什么要实地验证？

桌面研究的局限：
- 数据可以造假，但门店的人气造不了假
- 研报可以说好，但亲自站在店门口数最有说服力
- 产品好不好，自己用了才知道

段永平对FB的判断失误也来源于此：

> **"由于自己不用FB的任何产品，所以对FB的理解一直没达到很透彻的地步。"**

### 实地验证的层次

| 层次 | 具体行动 | 验证内容 |
|------|---------|---------|
| **第一层：产品体验** | 亲自使用公司的产品、服务 | 产品好不好？用户为什么喜欢？ |
| **第二层：门店观察** | 在门店外观察客流、客户画像、购买行为 | 谁在买？复购动机是什么？ |
| **第三层：用户对话** | 和真实用户/消费者沟通 | 他们为什么选这家不选竞争对手？ |
| **第四层：竞争对手** | 观察竞争对手的门店、产品、定价 | 替代品有多强？差异点是什么？ |

### 实地验证的段永平标准

> **"不大的门店生意极好，进店顾客绝大多数是本地成年人，华人占比仅一成左右。"**

这个观察回答了他最关心的几个问题：
1. 是不是中国人炒作？（→ 不是，本地人在买）
2. 是不是小孩子的东西？（→ 不是，成年人在消费）
3. 是不是一时的流行？（→ 一个门店的持续客流说明复购存在）

### 无法实地验证怎么办？

大多数情况下，你无法飞到一个城市去蹲点。替代方案：
- **产品体验**：能用的就用，能买的就买
- **用户评论**：看美团/亚马逊/App Store的真实用户评价
- **门店数据**：通过财报中的同店销售、坪效等指标间接判断
- **管理层讲话**：听创始人/CEO怎么谈论自己的产品和用户

## 七、迭代认知循环（最重要的新增）

### 7.1 为什么需要迭代？

段永平对泡泡玛特的判断经历了**9个月的认知迭代**：

```
2025年8月  → 看不懂10年后会怎样        → 不投资
2026年1月  → "无法理解人们为什么需要"  → 不投资
2026年3月  → 看到年报数据："居然这么好" → 重新审视
2026年4月  → 读传记+实地考察            → 开始建仓
2026年5月  → 理解创始人："和Jobs同级别" → ALL IN
```

**这不是一次性的"看懂了→买入"，而是一个认知逐步升级的过程。**

### 7.2 异常信号触发机制

当遇到**反直觉的信号**时，触发"第二遍分析"：

```
反直觉信号的定义：
  × 数据和你之前的判断方向相反
  × 数据好到你不信（"居然真的有这么好的盈利"）
  × 数据坏到你不信（好公司突然出问题）
  × 多次出现的同类信息（不同来源都在说同一件事）

触发动作：
  1. 不立刻否定信号，也不立刻相信
  2. 问自己：这个信号如果是真的，我的判断哪里错了？
  3. 去验证这个信号的真实性
  4. 如果信号为真 → 修正判断
  5. 如果信号为伪 → 坚持原有判断
```

### 7.3 认知升级的路径

段永平从"看不懂泡泡玛特"到"重仓"的路径可以抽象为：

```
第一遍：基于现有认知 → "看不懂" ✓（诚实面对自己）
    ↓
新信号出现（年报数据）→ 触发重新审视
    ↓
第二遍：收集新信息（读书、实地考察）
    ↓
产生类比突破："泡泡玛特=游戏化的服装公司"（用自己的经验连接新事物）
  （这个类比来自他玩《梦幻西游》买皮肤的经历——用自己的经验桥接新事物）
    ↓
重新审视商业模式："现在不需要想商业模式行不行得通了"
    ↓
最终决策
```

**关键洞察**：段永平不是靠"更仔细地分析数据"实现认知突破的。他是靠**用自己的经验去类比**——把泡泡玛特连接到他理解的东西（游戏皮肤）。

### 7.4 如何在分析中使用迭代

```
第一遍分析（快速判断）：
  → 结论：值得关注 / 不值得关注
  
如果值得关注：
  → 设置"认知升级触发器"
     • 等待下次财报
     • 等待某个关键产品发布
     • 等待管理层重大决策

第二遍分析（新信息来临时）：
  → 问自己：我的判断修正了吗？
  → 如果修正了：为什么？什么信息最关键？
  
第三遍分析（形成确定判断后）：
  → 买入 or 放弃
```

### 7.5 迭代式"看懂" vs 一次性"分析"

| 维度 | 一次性分析 | 迭代式认知 |
|------|-----------|-----------|
| 时间视角 | 一个时间点的快照 | 持续9-12个月的跟踪 |
| 信息收集 | 一次搜索研报 | 持续关注+实地验证 |
| 判断修正 | 难（第一印象锚定） | 容易（每次新信息修正一次） |
| 最终结论 | "分析完了，结论是X" | "我跟踪了1年，确信X" |
| 适合场景 | A型公司（生意自明） | B型公司（需要深度理解） |

## 八、环境变化检测——第一步不是"分析"，是"扫描颠覆性变化"（新增）

### 8.1 为什么要加这一层？

你之前的分析被指出"使用了过去的信息和观点"、"没有考虑大环境变化对商业模式的影响"。这是对的。

**问题是**：框架中所有分析维度（商业模式、企业文化、估值）都默认"过去的逻辑在未来仍然成立"。段永平本人对茅台或苹果的判断是在特定历史条件下做出的，这些条件可能已经变了。

腾讯在AI时代的案例说明了这个缺陷：
```
段永平说"腾讯确实是不错的公司"（2010-2020年代）
当时：微信是绝对的超级App，网络效应牢不可破
现在（2026年）：AI正在改写信息获取方式
→ 微信的封闭生态 → 在AI时代还是护城河吗？
→ 段永平当年的判断前提——"微信生态不可挑战"——动摇了
→ 所以不能直接套用他过去的结论
```

> **段永平自己的原则也支持"停下重检"：**
> **"发现是错的事情的时候要马上停止，不管多大的代价都是最小的代价。"**
>
> 如果环境变了，过去对的事情现在可能变成错的事情。**"发现错了"包括：发现过去的假设不成立了。**

### 8.2 颠覆性信号扫描（五维检测）

在进入任何分析之前（甚至在判断公司类型之前），先执行这个扫描：

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🌍 环境变化检测 — 扫描颠覆性信号
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【维度一：技术变革】
□ 过去3-5年这个行业有重大技术颠覆吗？
  如：AI对搜索/社交/内容分发的影响
□ 技术变革是"改良性的"（提高效率）还是"颠覆性的"（改变格局）？
  改良性 → 传统护城河可能仍然有效
  颠覆性 → 传统护城河可能被绕过
□ 现有玩家能利用新技术巩固优势，还是新玩家能利用新技术弯道超车？

【维度二：消费者行为变化】
□ 过去5-10年，用户的行为模式有没有根本性变化？
  如：年轻人还喝茅台吗？年轻人还用微信作为主要信息入口吗？
□ 如果有变化：是持续的趋势还是一时的潮流？
□ 这个变化对这个生意是利好还是利空？

【维度三：竞争格局变化】
□ 谁是新进入者？他们带来了什么不一样的模式？
  如：抖音/豆包在AI上的布局 vs 腾讯
□ 竞争是"在同一个维度上竞争"还是"换了竞争维度"？
  换维度是更危险的信号
□ 护城河对新人有效吗？（如：微信的网络效应对AI代理有效吗？）

【维度四：监管/政策变化】
□ 监管风向发生了什么变化？
  如：反垄断、数据安全、游戏版号、行业补贴
□ 这种变化是周期性的还是结构性的？

【维度五：全球化/供应链变化】
□ 全球化的方向变了？（出海受阻 vs 出海加速）
□ 供应链安全？地缘政治风险？

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
扫描结论：
  颠覆级别：□ 无颠覆 → 传统框架可以安全使用
           □ 有变化但改良性 → 框架可用，但在相关维度上要调整假设
           □ 有结构性颠覆 → 传统框架的适用性存疑，需要"框架外思考"
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### 8.3 "段永平说的前提条件"检查清单

知识库中段永平对每个公司的判断都是在特定条件下做出的。引用任何已有判断之前，必须完成这个检查：

```
段永平判断：________________（如"腾讯确实是不错的公司"）
引用出处：________________（如 02-商业模式.md、06-案例拆解.md）
最初判断的时间背景：________________（如 2018年，微信如日中天）

当前环境对比：
  技术条件     □ 没变 □ 变了（说明：______）
  竞争格局     □ 没变 □ 变了（说明：______）
  消费者行为   □ 没变 □ 变了（说明：______）
  监管环境     □ 没变 □ 变了（说明：______）

→ 结论：段永平的判断在当前环境下 □ 仍然有效 □ 需重新评估 □ 已不适用
```

**原则：** 
- 如果没有完成这个检查就引用段永平的过去判断 → **这是偷懒，不是分析**
- 如果环境变了但判断仍可能有效 → 需要说明"为什么即使环境变了仍有效"
- 如果环境变了且判断可能无效 → 诚实地说"段永平没有在这个环境下说过这个公司"

### 8.4 当发现颠覆性变化时怎么办？——"框架外思考"指引

如果颠覆性信号扫描得出"有结构性颠覆"的结论，**不要试图用现有框架硬套**。现有框架建立在对"好生意"的假设上，而颠覆性变化可能意味着这些假设已经过时。

这时应该做的是：

```
传统路径（不推荐）： 
  强行用框架分析 → 得出模糊结论 → 用"低估"来解释一切
  → 结果：买了腾讯在PE 15倍自以为"便宜"，却没看到AI对微信的侵蚀

推荐的"框架外思考"路径：
  ① 承认：传统框架可能不适用于当前环境
  ② 寻找新的分析维度：AI对用户行为的重塑、技术路线的选择权等
  ③ 设置观察信号：什么信息能验证或证伪我的担忧？
  ④ 用"小仓位"保持在场，等待认知升级
     （段永平在泡泡玛特上也是从"看不懂"开始，没有硬买）
```

### 8.5 段永平面对颠覆性变化的做法

段永平本人对颠覆性变化的态度值得模仿：

| 场景 | 段永平的做法 | 启示 |
|------|------------|------|
| 看不懂泡泡玛特（2025年） | **承认看不懂**，不买，"看不懂10年后会怎样" | 不硬套框架 |
| 年报数据反直觉（2026年3月） | **触发重新审视**，不固守"不投资"的结论 | 让数据反直觉时停下来 |
| 读传记+实地考察后（2026年4月） | **认知升级**，ALL IN | 环境变了，判断可以变 |
| FB面对抖音的威胁（2018年） | **判断自己没真懂**，卖出FB | 环境变了，原来的理解不够了 |
| 对汽车行业（比亚迪） | **诚实说不知道**，"我对汽车行业不熟悉" | 不在能力圈内就不说 |

**核心原则：**
- **环境变了，判断可以变。** 段永平自己对泡泡玛特的态度从"不投资"到"ALL IN"，证伪了这句话。
- **诚实面对"我不知道"。** 比"硬套框架给出一个错误的确定结论"好一万倍。
- **让反直觉信号触发重新审视，而不是忽略它。**

## 九、完整分析流程（九步法）

### Step 0：环境变化检测（新增前置步骤）

在开始任何公司分析之前，先执行颠覆性信号扫描。

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
□ 技术颠覆扫描完成    → 颠覆级别：无/改良性/结构性
□ 消费者行为变化扫描  → 变化级别：无/渐近/根本性
□ 竞争格局变化扫描    → 变化级别：无/加剧/换维度
□ 段永平判断的前提条件检查完成（如果引用了）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

如果扫描出"结构性颠覆" → 不要硬套框架
  走"框架外思考"路径：承认不确定性，用小仓位观察，等待认知升级
  详情见 第八章
```

> ⚠️ **环境变化检测是"框架的门"**——如果检测到结构性颠覆，传统框架可能不适用。
> 这时最诚实的结论是"我不知道"，而不是用框架硬套。

### Step 0.5：公司类型判断

```
□ 这家公司是生意优先型还是创始人优先型？
□ 判断依据是什么？
```

使用"公司类型判断"工具（见第二章）。

### Step 1：排除测试

4个问题，任何一个不通过直接跳过。

```
□ 这个生意我真正懂吗？
□ 生意模式一眼就能看出好吗？
□ 管理层有不本分迹象吗？
□ 有致命的Stop Doing违规吗？
```

### Step 2：生意叙事

写100-300字，讲清：它是什么生意、怎么赚钱、客户为什么选它。

### Step 3：深度分析（根据路径选择顺序）

**Path A（生意优先）：**
```
商业模式分析（最高权重）→ 企业文化诊断 → 价格/估值
```

**Path B（创始人优先）：**
```
创始人深潜（前置）→ 商业模式分析（重审）→ 企业文化 → 价格
```

### Step 4：实地验证

```
□ 我用过公司的产品吗？体验如何？
□ 我观察过它的门店/用户/竞争环境吗？
□ 我是否验证了分析中的关键假设？
```

### Step 5：综合判断

```
排除测试：□ 通过
生意叙事：□ 清晰
三维判断方向是否一致：□ 是 □ 否（不一致说明理解不够）
最终判断：□ 关注 □ 买入 □ 重仓 □ 不投
```

### Step 6：10年视角

```
□ 10年后护城河更宽还是更窄？
□ 如果跌一半，想买还是想跑？
□ "能睡好觉吗？"
```

### Step 7：进入迭代认知循环

```
□ 设置下次检视的时间点
□ 识别关键信号（什么数据会触发重新判断？）
□ 准备好修正认知
```

## 十一、反事实检验（强化版）

用段永平的终极问题检验自己的判断。

### 基础版（原框架已有）

> **"如果这家公司退市了，变成非上市公司，我以现在的市值全部买下来，我晚上能睡好觉吗？"**

### B型公司专属版

> **"如果创始人明天出了意外，我还会不会买这个公司？"**

- 答案是"会" → 公司可能是A型（造钟已完成），或者你对创始人的依赖被低估了
- 答案是"不会" → 确认是B型，这是正常的，但要意识到你在承担创始人风险

### 迭代检验版（新增）

> **"三个月后，什么信息会让我改变现在的判断？"**

提前设定期望，而不是等数据来了再被动反应。

| 如果我现在 | 我应该关注的关键信号 |
|-----------|------------------|
| 判断正确但不确定 | 下一个季度的数据能否验证我的逻辑 |
| 判断错误但没意识到 | 哪些反直觉信号会触发我重新检视 |
| 还没看懂的B型公司 | 创始人的访谈、传记、公开决策记录 |

## 十三、常见误区——更新版

### 误区一：所有公司用同一套模板

❌ 对所有公司都用"商业模式→文化→价格"线性套用

✅ 先判断公司类型，再选分析路径。B型公司从创始人入手，A型公司从模式入手。

### 误区二：创始人评估就是"企业文化打分"

❌ 创始人深潜 = 看CEO是否本分 + 是否造钟人

✅ 创始人深潜 = 传记研究 + 企业家共情 + 产品理念理解 + 商业直觉判断

### 误区三：分析一次就结束

❌ 分析完得出结论"可买/不可买"，不再跟踪

✅ 设置迭代循环：新信号触发重新审视，认知可以升级，"看不懂"可以是暂时的

### 误区四：桌面研究就够了

❌ 搜数据、读研报、看财报 = 懂了

✅ 真正的理解需要实地验证（门店、产品、用户），尤其是B型公司

### 误区五：忽略创始人年龄

❌ CEO评估只关注能力和道德

✅ 年龄决定"企业家复利"——段永平特别看重"他还能干25年以上"

### 误区六：默认"过去适用=未来适用"，忽略环境变化

❌ "段永平说过腾讯是好公司，所以腾讯现在还是好公司"
❌ 直接引用段永平的过去判断，没检查这些判断的前提条件还在不在

✅ **在引用任何段永平的过去判断之前，先做环境变化检测：**
   ```
   段永平说"腾讯确实是不错的公司"（约2018年）
   ```
